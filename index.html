1. 프로젝트 배경

기숙사 생활은 대학생들에게 낯선 환경 속에서 새로운 관계를 형성하는 중요한 공간입니다. 하지만 대부분의 룸메이트 배정이 랜덤으로 이루어지거나 지인 추천에 의존하고 있습니다. 이로 인해 생활 습관 차이로 인한 갈등이 빈번하게 발생하고 있습니다.

기존에는 이런 문제를 해결하거나 룸메이트를 선택할 수 있는 적합한 플랫폼이 없었고, 학생들은 이러한 상황을 받아들이는 것 외에는 별다른 선택지가 없었습니다. 학생들이 더 편안하고 조화로운 환경에서 생활할 수 있도록 새로운 디지털 솔루션이 필요했습니다.

2. 프로젝트 목적

학생들이 자신의 생활 습관과 성향에 맞는 룸메이트를 선택할 수 있도록 플랫폼을 구축하는 것이었습니다. 사용자들이 자신의 성향, 생활 습관, 선호도를 기반으로 룸메이트를 매칭하여 기숙사 생활에서의 불편함을 줄이고, 긍정적인 경험을 제공하여 학생들이 학업과 개인 생활에 더 집중할 수 있도록 돕는 데 중점을 두었습니다. 나아가 긍정적인 공동체 문화를 형성하고, 학생들 간의 새로운 연결을 지원하는 디지털 도구로 자리 잡는 것을 목표로 하고 있습니다.

1. 개발의 이유와 목적

코로나19 팬데믹은 전 세계적으로 사회적 상호작용의 방식을 급격히 변화시켰습니다. 정부의 사회적 거리두기 정책, 재택근무, 그리고 비대면 수업의 확산은 학생들 간의 물리적 상호작용 기회를 감소시켜, 우울증, 고립감, 그리고 소속감 결여와 같은 심리적 문제를 야기했습니다. 한국공학대학교 학생들 또한 예외는 아니었으며, 친구나 동료와의 만남 기회 부족으로 인해 사회적 고립감을 느끼는 경우가 많았습니다.

이러한 문제를 해결하고자, 학생들 간의 상호작용을 촉진하는 블라인드 매칭 웹 서비스를 개발하게 되었습니다. 이 서비스는 학생들이 자신의 프로필(예: MBTI, 취미, 특기)을 기반으로 적합한 상대를 추천받아 소통할 수 있는 환경을 제공합니다. 이를 통해 새로운 인맥을 형성하고, 사회적 고립감을 해소하며, 학교 내 소속감을 강화하는 것을 목표로 하고 있습니다.

2. 데이터베이스 설계

블라인드 매칭 서비스는 사용자 정보와 매칭 데이터를 효율적으로 관리하기 위해 체계적인 데이터베이스 설계가 필요합니다. 주요 테이블은 다음과 같습니다: 사용자(User) 테이블: 기본 사용자 정보를 저장합니다. 인증(Authentication) 테이블: 학교 소속 여부를 확인합니다. 내 프로필(MyProfile) 테이블: 사용자의 성향과 특기를 저장합니다. 선호 프로필(PreferredProfile) 테이블: 사용자가 선호하는 상대의 정보를 저장합니다. 매칭 결과(MatchingResult) 테이블: 매칭된 사용자 정보를 기록합니다. 이러한 데이터베이스 설계는 사용자 정보를 체계적으로 관리하고 매칭 알고리즘의 효율적인 작동을 지원합니다.

3. 개념적, 논리적, 시스템적 설계

1) 개념적 설계: 블라인드 매칭 서비스는 학생들 간의 교류를 증진하고, 사회적 고립감을 해소하기 위해 설계된 플랫폼입니다. 사용자 프로필과 선호 정보를 기반으로 매칭을 제공하여 의미 있는 만남을 지원합니다.

2) 논리적 설계: 사용자 프로필과 선호 정보를 비교하여 매칭 점수를 계산하며, 데이터베이스를 통해 효율적으로 데이터를 관리합니다. 데이터 항목 간 관계를 명확히 하여 무결성을 보장합니다.

3) 시스템 인터페이스 설계: 직관적이고 간단한 사용자 인터페이스를 제공하며, HTML, CSS, JavaScript를 활용한 프론트엔드와 Node.js 기반 백엔드, MySQL 데이터베이스를 연동합니다. RESTful API를 통해 클라이언트와 서버 간 통신을 지원하며, 데이터 보안을 위한 HTTPS와 암호화된 인증 체계를 도입합니다.

4. 매칭 알고리즘

매칭 알고리즘은 사용자 프로필과 선호 프로필을 비교하여 매칭 점수를 계산합니다. 성별, MBTI, 취미, 특기 항목의 일치 여부에 따라 점수를 부여하며, 점수가 높은 사용자끼리 매칭됩니다. 이를 통해 사용자는 자신의 성향에 맞는 상대와의 만남을 기대할 수 있습니다.

5. 목표

기술적 목표: 웹 서비스 개발 전반에 대한 실질적인 경험을 쌓고, 데이터 관리 및 보안 체계 구축을 통해 웹 개발 능력을 향상시키는 것이 주요 목표입니다.

사용자 목표: 서비스를 통해 500명의 사용자가 연결되고 소통할 수 있는 플랫폼으로 성장하는 것을 목표로 하며, 이를 위해 지속적인 피드백 반영과 기능 개선을 추진합니다.

6. ERD 설계

ERD(Entity-Relationship Diagram)는 데이터베이스 내 테이블 간의 관계를 시각적으로 표현하여 데이터 구조를 명확히 이해할 수 있도록 돕습니다. 블라인드 매칭 서비스의 주요 테이블과 관계는 다음과 같습니다: 사용자(User) 테이블은 인증(Authentication), 내 프로필(MyProfile), 선호 프로필(PreferredProfile)과 1대1 관계를 가집니다. 인증(Authentication) 테이블은 사용자 정보의 학교 소속 여부를 확인하는 데 사용됩니다. 매칭 결과(MatchingResult) 테이블은 여러 사용자 정보를 저장하며, 사용자와 N:N 관계를 형성합니다. 이러한 설계를 통해 데이터의 무결성을 유지하고, 효율적인 데이터 조회 및 매칭 결과 제공이 가능하도록 지원합니다.

7. 시스템 구현 계획

프론트엔드: HTML, CSS, JavaScript를 활용해 사용자 인터페이스를 설계합니다. 사용자가 쉽게 정보를 입력하고 결과를 확인할 수 있도록 직관적인 화면을 제공합니다. 주 기능은 회원가입 및 로그인 화면, 프로필 작성 및 선호 프로필 설정 화면, 매칭 결과 확인 화면입니다.

백엔드: Node.js를 기반으로 서버 로직을 개발합니다. 주요 기능은 데이터베이스 연동, 인증 처리, 매칭 알고리즘 실행 등이 포함됩니다.

데이터베이스: MySQL을 활용하여 데이터 저장소를 설계 및 구현합니다. 데이터 무결성과 보안을 고려하여 데이터를 체계적으로 관리합니다.

API 설계: RESTful API를 통해 프론트엔드와 백엔드 간의 데이터 통신을 지원합니다. 주요 API는 사용자 등록 및 인증 API, 프로필 입력 및 조회 API, 매칭 결과 조회 API입니다.
